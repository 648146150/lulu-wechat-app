# 工具箱升级 · 整体实施方案

## 一、从「记账本」到「工具箱」

```
升级前                          升级后

app.json                        app.json
  记账本 3 页面                   仪表盘（首页 · 唯一下方入口）
                                ├── 记账本（3 子页面）
                                ├── 总资产（新建）
                                ├── 步数（新建 · 云开发）
                                ├── 经期记录（新建）
                                └── 自定义/设置（卡片槽位管理）
```

Dashboard 底部仅一个「首页」入口，其余模块从卡片点击进入，利用系统导航栏返回。

## 二、页面路由设计

```
仪表盘 (pages/dashboard/dashboard)  ← 唯一底栏入口
    │
    ├── 点记账卡片 → pages/bookkeeping/index/index
    │                   ├── 记账页 add
    │                   └── 账单列表 history
    │
    ├── 点资产卡片 → pages/assets/assets
    │                   └── 添加/编辑账户（底部弹窗表单）
    │
    ├── 点步数卡片 → pages/steps/steps
    │                   └── 微信运动解密（云函数 getWeRunData）
    │
    ├── 点经期卡片 → pages/period/period
    │                   └── 日历标记 + 周期预测
    │
    └── 点自定义 → pages/settings/settings
                        └── 卡片槽位管理（替换/增删/上限6）
```

## 三、卡片网格设计

| 卡片数 | 布局 | 说明 |
|:--:|------|------|
| 1-4 | 2列 × N行 | 默认 4 槽位：记账本·总资产·步数·自定义 |
| 5-6 | 3列 × 2行 | 用户添加第 5/6 个槽位后自动切换 |

- 自定义卡片永久保留，不可替换、不可删除
- 其他卡片长按可替换
- 设置页支持：添加槽位（≤6）· 替换工具 · 删除槽位
- 达到 6 上限后只能替换，不可新增

## 五、技术要点

| 事项 | 处理 |
|------|------|
| 天气 API Key | `utils/config.js`，使用高德 Web 服务 Key |
| 城市名数组兼容 | `formatCity` 处理高德返回数组/字符串两种类型 |
| 步数解密 | 云函数 `getWeRunData` 调用 `cloud.openapi.werun`（需发布） |
| 存储 | 全部使用 `wx.Storage` 本地存储，无后端 |
| 卡片配置 | `dashboard_slots` key 存储用户槽位配置 |
| CSS 网格 | CSS Grid `grid-template-columns` 动态 2/3 列 |
